DisplayHelper Pro 개인정보 처리방침

시행일: 2026년 6월 2일

DisplayHelper Pro는 미러링된 Mac 설정에서 내장 디스플레이를 제어하고, 선택적으로 Mac이 더 강한 저장된 Wi-Fi로 전환하도록 돕는 로컬 우선 macOS 메뉴 막대 유틸리티입니다.

이 번역은 편의를 위해 제공됩니다. 이 정책의 공식 버전은 영어 버전입니다.

앱 계정 없음 DisplayHelper Pro 로그인이나 사용자 프로필이 없습니다.
분석 없음 이벤트 추적, 세션 추적, 광고 픽셀 또는 사용 텔레메트리가 없습니다.
Wi-Fi 비밀번호 없음 Wi-Fi 전환은 macOS 네트워크 정보를 사용하며, DisplayHelper Pro는 비밀번호를 읽지 않습니다.

데이터 수집

DisplayHelper Pro는 디스플레이 관리 기능을 통해 개인 정보를 수집, 저장, 판매, 공유 또는 전송하지 않습니다. 사용자가 지원 이메일을 보내거나 라이선스를 활성화하는 경우를 제외하면, Xintech LLC는 앱으로부터 디스플레이, 설정, 사용 현황 또는 기기 정보를 받지 않습니다.

  • DisplayHelper Pro 계정은 필요하지 않습니다.
  • 분석, 추적, 광고 또는 충돌 보고 SDK가 포함되어 있지 않습니다.
  • 백그라운드 텔레메트리가 Xintech LLC로 전송되지 않습니다.

로컬 데이터

DisplayHelper Pro는 Mac에 소량의 로컬 환경설정과 제한된 로컬 진단 로그를 저장할 수 있습니다. 이는 실행 간 동작을 일관되게 유지하고, 사용자가 버그를 신고하기로 선택했을 때 이메일 초안을 준비하는 데에만 사용됩니다.

로컬 환경설정에는 다음이 포함될 수 있습니다.

  • 자동 블랙아웃 활성화 여부.
  • 실행 시 내장 미러링을 활성화할지 여부.
  • DisplayHelper Pro 메뉴 동작을 위한 사용자 지정 키보드 단축키.
  • Auto Wi-Fi Switch 활성화 여부와 신호 강도 임계값.
  • 선호 외부 디스플레이를 기억하기 위한 로컬 디스플레이 식별자.
  • 버그 보고서 초안에 사용되는 제한된 DisplayHelper Pro 진단 메시지 로그.
  • 체험 시작 및 만료를 관리하기 위한 로컬 및 키체인 표시자.
  • 라이선스 활성화 시 Mac 키체인에 저장되는 라이선스 키와 Lemon Squeezy 라이선스 인스턴스 식별자.
  • 오프라인 유예 동작에 사용되는 마지막 성공 검증 시간 등 비밀이 아닌 라이선스 타임스탬프.

앱은 macOS 디스플레이 API를 사용하여 디스플레이 배치와 미러링을 확인하고, 밝기와 임시 어둡게를 제어하며, 로그인 시 시작을 관리하고 필요한 경우 안전한 밝기 값을 복원합니다. 임시 어둡게 중에는 디스플레이를 복원하기 위해 키가 눌렸는지만 확인하며, 키 이름, 입력된 텍스트 또는 키보드 기록을 저장하지 않습니다.

Wi-Fi 상태 및 위치 권한

Auto Wi-Fi Switch는 선택 사항입니다. 활성화되면 DisplayHelper Pro는 공개 macOS CoreWLAN API를 사용하여 현재 Wi-Fi 이름, 액세스 포인트 주소, 신호 강도와 대역을 읽고, 더 나은 대안이 될 수 있는 저장된 자동 연결 네트워크를 검색합니다.

macOS는 앱이 Wi-Fi 네트워크 이름과 액세스 포인트 주소를 보기 전에 위치 권한을 요구할 수 있습니다. DisplayHelper Pro는 메뉴와 Auto Wi-Fi Switch에 필요한 Wi-Fi 정보에만 이 권한을 사용합니다. 앱은 위치 기록을 수집하지 않고, Wi-Fi 정보를 Xintech LLC로 보내지 않으며, Wi-Fi 비밀번호를 읽거나 저장하거나 전송하지 않습니다.

네트워크 활동

DisplayHelper Pro의 일반 디스플레이 제어에는 네트워크 연결이 필요하지 않습니다. 자동 업데이트 확인이 켜져 있거나 사용자가 “업데이트 확인...”을 선택하면 앱이 공개 Sparkle 업데이트 피드에 접속할 수 있습니다.

라이선스 작업은 라이선스를 활성화, 검증 또는 비활성화하기 위해 Lemon Squeezy에 접속합니다. 요청에는 라이선스 키와 로컬 Mac 이름을 기반으로 한 인스턴스 이름이 포함되어, 나중에 해당 Mac을 식별하고 비활성화할 수 있습니다. 라이선스 요청은 분석, 광고 또는 사용 추적에 사용되지 않습니다.

버그 신고, 기능 요청 및 피드백

“버그 신고...”, “기능 제안...”, “피드백 보내기...” 메뉴 항목은 기본 이메일 앱에서 Xintech LLC로 보내는 초안을 엽니다. 앱은 이메일을 자동으로 보내지 않습니다. 보내기 전에 초안을 확인, 수정 또는 삭제할 수 있습니다.

버그 신고 초안에는 앱 버전, macOS 버전, 프로세서 아키텍처, Auto Blackout 상태, 미러링 상태, 현재 앱 상태와 함께 제한된 최근 DisplayHelper Pro 진단 로그 발췌가 포함될 수 있습니다.

진단 및 로그

DisplayHelper Pro는 Apple의 로컬 통합 로깅 시스템에 진단 메시지를 기록하고 작은 앱 로컬 진단 로그를 유지합니다. 이러한 로그는 Mac에 로컬로 저장됩니다. 사용자가 이메일 초안에서 직접 보내기를 선택하지 않는 한 아무것도 전송되지 않습니다.

사용자의 선택

  • 피드백 초안을 보내지 않거나, 수정하거나, 진단 세부 정보를 삭제할 수 있습니다.
  • Sparkle 업데이트 환경설정에서 자동 업데이트 확인을 끌 수 있습니다.
  • DisplayHelper Pro 메뉴에서 Auto Wi-Fi Switch를 켜거나 끌 수 있습니다.
  • macOS 시스템 설정에서 Wi-Fi 정보를 위한 위치 권한을 허용, 거부 또는 철회할 수 있습니다.
  • 체험 기간 동안 라이선스를 활성화하지 않고 Pro 디스플레이 제어를 사용할 수 있습니다.
  • 라이선스 창에서 이 Mac을 비활성화할 수 있습니다.
